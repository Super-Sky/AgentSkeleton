# AGENTS.md

语言版本：

- [English](AGENTS.md)
- [中文](AGENTS.zh-CN.md)

这个仓库是一个 AI 友好的脚手架项目。agent 在本仓库中的工作应优先保证结构清晰、约束显式、规则可复用。

## 工作原则

- 优先遵守共享项目约定，而不是依赖某个工具的私有习惯。
- 将 CLI 视为主要产品形态。
- 将模板视为仓库拥有并维护的核心资产。
- 除非确有必要，否则保持 Codex 与 Claude Code 的支持策略一致。
- 优先选择易读、易理解的仓库结构，而不是炫技式自动化。
- 默认将本仓库视为 AI 编写，而不是手工编码项目。

## 仓库意图

本项目的存在是为了帮助用户：

- 用 agent 友好的结构启动新项目
- 将已有项目迁移为 agent 友好的结构
- 生成 agent 协作说明文件
- 检查仓库结构与规范缺失项

## 文档规则

- 产品定位写在 `README.md`。
- 更深入的产品决策写在 `docs/`。
- agent 专属说明保持最小且明确。
- 不要依赖未写入仓库的隐性约定。

## 实现方向

- 主语言：Go
- 主要入口：CLI
- 共享脚手架资产：`templates/`
- 实现代码：`cmd/` 和 `internal/`

## 第一阶段预期

第一阶段应优先完成仓库定义、目录结构和文档，而不是立即实现完整 CLI。

## 编写约束

本项目预期通过 AI agent 完成实现。

- 人类负责定义方向、标准和验收条件。
- agent 负责生成和修改仓库内容。
- 手工编写源码不是这个项目的默认工作方式。
