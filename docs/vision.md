# Vision

AgentSkeleton aims to become a practical foundation for AI-assisted software construction and maintenance.

The long-term idea is simple:

- users interact with a CLI
- the CLI applies and maintains repository templates
- the generated projects are easy for both humans and agents to understand
- the project itself is authored through AI collaboration rather than manual coding

## Design Principles

- Explicit beats implicit.
- Shared structure beats agent-specific fragmentation.
- Stable defaults beat excessive early configuration.
- Migration matters as much as greenfield setup.
- AI-authored workflow beats manual implementation drift.

## Product Shape

The repository should contain:

- template assets that define the scaffold
- documentation that explains the rules
- CLI code that applies those rules

## Why This Matters

Most projects are not built with AI collaboration in mind. Important context often lives in people, not in the repository. AgentSkeleton exists to make that context visible, structured, and portable.
