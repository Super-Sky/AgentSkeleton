# Vision

AgentSkeleton aims to become a practical foundation for AI-assisted documentation architecture.

The long-term idea is simple:

- users interact with a CLI
- the CLI guides documentation conversations and organizes context
- large models draft the actual repository documents
- the resulting repositories are easier for both humans and agents to understand

## Design Principles

- Explicit beats implicit.
- Shared structure beats agent-specific fragmentation.
- Stable defaults beat excessive early configuration.
- Documentation reshaping matters as much as greenfield setup.
- Guided collaboration beats ad hoc prompting.

## Product Shape

The repository should contain:

- documentation blueprints that define target repository structure
- documentation that explains the guidance model
- CLI code that runs the guidance flow

## Why This Matters

Most projects are not built with AI collaboration in mind. Important context often lives in people, not in the repository. AgentSkeleton exists to make that context visible, structured, and portable.
