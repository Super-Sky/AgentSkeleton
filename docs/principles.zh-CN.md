# Principles

## 基线原则

AgentSkeleton 由一组不可轻易动摇的产品原则定义。

## AI-First 编写方式

- 生产代码默认应由 AI agent 生成。
- 人类负责方向、约束、评审和验收。
- 手工编码不是本仓库的默认工作模式。

## 双 Agent 支持

- Codex 和 Claude Code 都是一等工作流。
- 仓库应尽可能保持一套共享结构。
- 只有在确有真实差异时，才引入 agent 专属文件。

## 模板驱动的产品模型

- 模板不是附属资产，而是产品核心。
- CLI 的职责是生成、检查和维护这些资产。
- 仓库结构应当清楚反映这一产品模型。

## 显式仓库规则

- 重要上下文必须沉淀在仓库中，而不是停留在私人记忆里。
- 命名、目录和工作流约束都应文档化。
- 隐性约定本身就是缺陷。

## 面向外部可用的质量标准

- 项目应当能够被外部用户使用，而不只是内部团队自用。
- 内部使用应受益于同一套面对外部用户所需的清晰度。
- 早期版本仍应保持范围克制，并坚持强默认值。
