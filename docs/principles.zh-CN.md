# Principles

## 基线原则

AgentSkeleton 由一组不可轻易动摇的产品原则定义。

## AI-First 协作方式

- 仓库文档默认应由大模型起草。
- 人类负责方向、约束、评审和验收。
- 产品的职责是引导文档建设，而不是生成业务代码。

## 双 Agent 支持

- Codex 和 Claude Code 都是一等工作流。
- 仓库应尽可能保持一套共享结构。
- 只有在确有真实差异时，才引入 agent 专属文件。

## 文档蓝图驱动的产品模型

- 文档蓝图不是附属资产，而是产品核心。
- CLI 的职责是引导、检查和维护这些资产。
- 仓库结构应当清楚反映这一产品模型。

## 结构策略

- 新项目可以使用推荐的仓库模式。
- 对于应用型新项目，优先推荐 `internal/app` 导向的结构。
- 默认不应强迫已有项目迁移到新的代码布局。
- 对老项目，产品应先帮助解释和文档化现有结构，再决定是否建议结构调整。

## CLI 核心、宿主协作的交付方式

- 产品核心是 CLI。
- Codex 和 Claude Code 是主要宿主环境，而不是产品本体。
- CLI 应输出宿主模型可直接消费的结构化引导。

## 显式仓库规则

- 重要上下文必须沉淀在仓库中，而不是停留在私人记忆里。
- 命名、目录和工作流约束都应文档化。
- 隐性约定本身就是缺陷。

## 面向外部可用的质量标准

- 项目应当能够被外部用户使用，而不只是内部团队自用。
- 内部使用应受益于同一套面对外部用户所需的清晰度。
- 早期版本仍应保持范围克制，并坚持强默认值。
