# Feature: CLI Workflow

## Purpose

This feature area covers the current CLI workflow responsibilities.

## Current Scope

The CLI currently supports:

- initializing new documentation context
- reshaping legacy repository documentation context
- producing workflow plans
- producing drafting packages
- validating and applying structured answers
- inferring safe structural refreshes for existing context
- reporting version metadata

## Current Stable Surface

The current stable surface is:

- `init-docs`
- `reshape-docs`
- `plan`
- `focus-doc`
- `response --apply`
- `version`

The current helper surface also includes:

- `update`
