# Vision

AgentSkeleton 的目标，是成为 AI 辅助软件构建与维护的实用基础设施。

长期形态很简单：

- 用户通过 CLI 与项目交互
- CLI 负责应用和维护仓库模板
- 生成出来的项目既便于人类理解，也便于 agent 理解
- 项目本身通过 AI 协作演进，而不是依赖手工编码

## 设计原则

- 显式优于隐式。
- 共享结构优于按 agent 分裂。
- 稳定默认值优于过早暴露大量配置。
- 历史项目迁移与新项目初始化同样重要。
- AI 编写工作流优于手工实现带来的漂移。

## 产品形态

仓库应当包含：

- 定义脚手架的模板资产
- 解释规则的文档
- 应用这些规则的 CLI 代码

## 为什么要做这件事

大多数项目在创建时并没有按 AI 协作方式组织。很多关键上下文留在人脑中，而不是沉淀在仓库里。AgentSkeleton 的目标，就是让这些上下文变得可见、结构化、可迁移。
