# Vision

AgentSkeleton 的目标，是成为 AI 辅助文档架构建设的实用基础设施。

长期形态很简单：

- 用户通过 CLI 与项目交互
- CLI 负责引导文档对话并组织上下文
- 大模型负责真正撰写仓库文档
- 最终形成的仓库既便于人类理解，也便于 agent 理解

## 设计原则

- 显式优于隐式。
- 共享结构优于按 agent 分裂。
- 稳定默认值优于过早暴露大量配置。
- 历史项目文档重塑与新项目初始化同样重要。
- 引导式协作优于临时性提示词堆叠。

## 产品形态

仓库应当包含：

- 定义目标文档结构的蓝图
- 解释引导方法的文档
- 运行引导流程的 CLI 代码

## 为什么要做这件事

大多数项目在创建时并没有按 AI 协作方式组织。很多关键上下文留在人脑中，而不是沉淀在仓库里。AgentSkeleton 的目标，就是让这些上下文变得可见、结构化、可迁移。
