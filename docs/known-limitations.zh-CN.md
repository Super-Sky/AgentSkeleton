# 已知限制

## 目的

本文档用于记录 AgentSkeleton 当前公开发布路径中的已知限制。

对于 `v0.1.0`，这些限制应被视为显式范围边界，而不是无意遗漏。

## 当前限制

- 官方宿主接入目前优先采用 skill 形态，而不是完整 plugin 产品。
- 零命令行为已经完成规范与骨架，但仍需要宿主验证记录来证明它真正成立。
- 当前稳定宿主契约被有意收窄，集中在：
  - `init-docs`
  - `reshape-docs`
  - `plan`
  - `focus-doc`
  - `response --apply`
- `workflow`、prompt 格式细节以及更广泛的便利表面，当前还不应作为稳定宿主依赖路径。
- 当前 release packaging 主要覆盖 CLI 构建产物，还不是面向插件市场级别的宿主分发。
- 除当前构建矩阵外，仓库尚未声明广泛跨平台支持承诺。
- 在 `v0.1.0` 被视为真正可发布之前，仍需要补齐宿主验证证据。
- 项目依然聚焦于仓库文档编排，而不是业务代码生成。

## 当前阶段非目标

- 重型插件生态
- 丰富的 UI 管理界面
- 超出当前有界重试路径之外的高级自动修复循环
- 深度语言级项目脚手架
- 在没有用户澄清的情况下全自动完成整个仓库

## 指引

如果某个工作流依赖于稳定协议核心之外的行为，那么在有明确文档之前，都应视为 provisional 或 experimental。
