# Roadmap

## Phase 1

- 明确项目定位
- 明确命名约定
- 明确 Codex 与 Claude Code 的支持策略
- 建立基础仓库结构
- 起草 CLI 命令方向

## Phase 2

- 实现首版 Go CLI 骨架
- 增加 `init-docs`、`reshape-docs`、`plan`、`next` 命令骨架
- 增加蓝图加载与引导流程

## Phase 3

- 增加仓库文档诊断能力
- 增加已有项目文档重塑辅助能力
- 增加常见项目状态下的对话流程

## Phase 4

- 提升可扩展性
- 验证在宿主 agent 环境中的外部使用方式
- 强化适用于更广泛场景的默认值
