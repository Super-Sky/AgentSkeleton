# Architecture Notes

## 核心模型

AgentSkeleton 有两个核心层：

- 存储在仓库中的模板资产
- 负责应用、生成和校验这些资产的 CLI

这个仓库预期通过 AI 生成的改动持续演进，人类更像产品负责人、评审者和约束定义者，而不是主要代码作者。

## 支持模型

项目应通过以下方式同时支持 Codex 与 Claude Code：

- 一套共享结构
- 一套共享文档模型
- 仅在需要时补充少量 agent 专属文件

## 初始目录意图

- `cmd/`：CLI 入口
- `internal/`：内部实现包
- `templates/`：脚手架资产与文件模板
- `examples/`：生成结果示例或参考项目
- `docs/`：产品与设计文档
