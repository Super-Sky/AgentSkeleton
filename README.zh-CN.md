# AgentSkeleton

语言版本：

- English: `README.md`
- 中文: `README.zh-CN.md`

AgentSkeleton 是一个 AI-first 的项目脚手架，用于帮助团队以 agent 友好的方式构建和维护软件项目。

它围绕一个简单模型展开：

- 模板是基础资产
- CLI 是主要用户入口
- 默认支持 Codex 与 Claude Code 两种协作模式

它主要服务于两类场景：

- 初始化新项目，直接建立清晰的 agent 友好结构
- 将历史项目迁移为更适合 AI 工具与人类协作维护的结构

仓库内维护的模板和基础资产，是 CLI 用来生成、检查和维护项目结构的核心基础。

## 它是什么

AgentSkeleton 不是业务系统，而是一个项目构建与维护脚手架。

它的目标，是让项目对人类和 AI agent 都更容易理解、更安全演进，也更容易长期维护。

## 项目目标

- 提供一套可复用的项目模板，作为脚手架基石。
- 交付一个可执行的 CLI，作为用户主要入口。
- 同时支持 Codex 风格 agent 工作流与 Claude Code 工作流。
- 在两种 agent 模式之间尽量共享核心结构，只保留最小必要差异。
- 同时支持新项目初始化和历史项目 AI 迁移。
- 保持项目以 AI 生成为主，而不是手工编写源码。

## 核心原则

- AI-first 编写方式
- 在不同 agent 模式之间共享结构
- 仓库规则显式化
- 先有稳定默认值，再谈重度定制
- 既支持新项目，也支持历史项目迁移

## 第一阶段的非目标

- 一开始就覆盖所有项目类型的自动化。
- 一开始就支持很多语言和框架的深度生成器。
- 在核心流程尚未验证前就引入复杂插件体系。

## MVP 范围

第一阶段重点是先把定义和结构建立起来。

- 新仓库初始化。
- 为已有仓库提供 AI 迁移指导。
- 生成协作文件，例如 `AGENTS.md` 和 `CLAUDE.md`。
- 扫描仓库结构并给出基础诊断。

## 支持的 Agent 模式

AgentSkeleton 默认支持：

- Codex / agent mode
- Claude / Claude Code mode

支持方式采用：

- 一套共享核心结构
- 一套共享模板基础
- 在确有必要时补充少量 agent 专属说明文件

这样可以控制维护成本，避免演化成两套互相分裂的项目体系。

## 编写原则

AgentSkeleton 期望通过 AI 协作完成构建，而不是通过手工写代码推进。

默认前提是：

- 不期望存在手写的生产源码
- 仓库改动主要由 AI agent 提议并实现
- 人类负责定义目标、约束和验收标准

这条原则会直接影响产品设计和仓库协作方式。完整基线见 `docs/principles.zh-CN.md`。

## 规划中的 CLI 方向

CLI 预期会成为用户的主要入口。首批命令方向包括：

- `init`：基于脚手架初始化新项目
- `migrate`：将已有项目迁移到脚手架结构
- `generate`：生成 agent 协作文件
- `scan`：扫描仓库结构并报告缺失项

## 当前状态

仓库目前处于初始定义阶段。第一次推送应当至少建立：

- 核心文档
- 基础目录结构
- 命名约定
- agent 支持策略
- CLI 方向

## 贡献方向

在早期阶段，所有决策都应优先选择清晰、显式、默认稳定的方案，而不是过早追求高度灵活性。
